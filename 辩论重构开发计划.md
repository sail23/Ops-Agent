# 辩论系统重构开发计划

## 目标

将辩论从"编排器驱动的固定模板"改为"完全由 LLM 驱动的开放辩论"。

### 核心原则

1. **LLM 的 `next_turn` 是无条件指令** —— 编排器不否决、不覆盖、不修正
2. **所有 Agent 发言对全体可见** —— 每轮输出写入 SharedBoard，所有人可读
3. **唯一安全阀是 `max_turns`** —— 移除所有硬编码阈值（review_rounds、converge_attempts、debate_cycles）

---

## 步骤 1：`parse_next_turn` 默认值改为保持当前发言者（而非 TERMINATE）

**文件**：`debate.py`

**改动**：

```python
# 当前
return NextTurnHint.TERMINATE

# 改为
logger.warning("LLM output has no next_turn directive, continuing current speaker")
# 让编排器保留当前 role 作为 next_role，而不是终止辩论
```

**理由**：`parse_next_turn` 默认返回 TERMINATE 是早期辩论经常 4 轮结束的根本原因。当 LLM 没有按格式输出指令时，应该保持当前节奏而不是强制终止。

**风险**：低。如果不设置 fallback 值，需要让调用方（路由）处理缺失 case。

---

## 步骤 2：移除路由中所有否决 LLM 的硬编码规则

**文件**：`debate_orchestrator.py`

### 2a：移除 `review_rounds` 强制 CONVERGE

```diff
- # 防止无限 review 循环：连续 5 轮 review 后强制进入 converge
- if next_hint in (NextTurnHint.OPS, NextTurnHint.SRE, NextTurnHint.CODE):
-     if self._state.review_rounds >= 5:
-         ...
-         continue
```

移除后，LLM 说 `code` 就去 Code，说 `sre` 就去 SRE，不限次数。

### 2b：移除 `debate_cycles` 否决 CONVERGE

```diff
- elif not enough_cycles:
-     next_role = DebateRole.SRE
```

移除后，LLM 说 `converge` 就去执行收敛判断，不管回环够不够。

### 2c：移除 `converge_attempts ≥ 5` 强制 Report

```diff
- elif self._state.converge_attempts >= 5:
-     ...
-     next_role = DebateRole.REPORT
```

### 2d：移除非 Report 角色 `terminate` 强制 Report

```diff
- elif next_hint == NextTurnHint.TERMINATE:
-     if role == DebateRole.REPORT:
-         ...
-     else:
-         next_role = DebateRole.REPORT
```

改为：任何角色输出 TERMINATE 都立即终止辩论。如果 LLM 觉得可以结束，就让它结束。

**风险**：中。移除安全阀后需要确认 `max_turns` 是唯一的兜底机制。

## 步骤 3：简化终止逻辑，仅保留 `max_turns`

**文件**：`debate_orchestrator.py`、`debate.py`

### 3a：从 `DebateState` 移除不再需要的字段

移除：
- `has_ops_review` / `has_sre_review` / `has_code_review`
- `converge_attempts`
- `debate_cycles`
- `await_code_revision`
- `review_rounds`
- `has_ops_initial` / `has_sre_initial` / `has_code_initial`（可选，round name 已包含信息）
- `agent_confidence`（可选，不参与路由决策）
- `_update_convergence()`（收敛分不再用于路由）

保留：
- `max_turns`（唯一安全阀）
- `terminated` / `termination_reason`

### 3b：简化 `_should_terminate()`

```python
def _should_terminate(self) -> bool:
    if self._state is None:
        return True
    if self._state.terminated:
        return True
    if self._state.current_turn >= self._state.max_turns:
        self._state.terminated = True
        self._state.termination_reason = "max_turns"
        return True
    return False
```

移除 `max_turns + 3` 的宽限逻辑。

### 3c：`_get_round_for_role()` 改为仅判断初始/审视

不再依赖 `has_*_initial` 标记，改为检查 turns 中是否已有该角色的发言记录：

```python
def _get_round_for_role(self, role: DebateRole, state: DebateState) -> DebateRound:
    """根据角色在辩论中是否已发言决定初始轮/审视轮。"""
    has_initial = any(t.role == role.value and t.round.endswith("_initial") for t in state.turns)
    if not has_initial:
        return DebateRound.for_role_initial(role)  # OPS_INITIAL / SRE_INITIAL / CODE_INITIAL
    return DebateRound.for_role_review(role)       # OPS_REVIEW / SRE_REVIEW / CODE_REVIEW
```

**风险**：低至中。移除状态跟踪后代码更简洁，但需要确保 `_get_round_for_role` 仍能正确分配轮次。

## 步骤 4：Agent 输出写入 SharedBoard，实现全局可见

**文件**：`debate_orchestrator.py`、`debate_board.py`

### 4a：每轮辩论结果写入 SharedBoard

在 `run_debate` / `stream_debate` 的每轮循环末尾，追加到 SharedBoard：

```python
# 每轮结束后，将本轮内容写入 SharedBoard
self._shared_board.set(f"debate_turn_{self._state.current_turn}", {
    "round": round_obj.value,
    "role": role.value,
    "agent_id": agent.agent_id,
    "content": conclusion,
    "reasoning": reasoning,
    "next_turn": next_hint.value,
})
self._shared_board.set("debate_latest", {
    "turn": self._state.current_turn,
    "role": role.value,
    "content": conclusion,
})
```

### 4b：Agent 的 user prompt 从 SharedBoard 读取全部历史

修改 `_build_user_prompt()`：

```python
def _build_user_prompt(self, ctx, board, round_obj):
    lines = [...故障信息...]
    
    # 从 SharedBoard 读取所有历史辩论内容
    history = self._shared_board.snapshot()
    debate_keys = [k for k in history if k.startswith("debate_turn_")]
    if debate_keys:
        lines.append("\n## 辩论全程记录")
        for key in sorted(debate_keys):
            turn_data = history[key]
            lines.append(f"\n### {turn_data['role']} ({turn_data['round']}):")
            lines.append(turn_data['content'][:1000])
    
    return "\n".join(lines)
```

### 4c：`DebateBoard` 降级为仅用于历史记录/调试（或移除）

如果所有数据都存在于 SharedBoard 中，`DebateBoard` 可作为只读的历史视图保留，或完全移除。

**风险**：中。SharedBoard 是单例，需要确保并发辩论（如果有）不冲突。

## 执行顺序

```
步骤 6 (修复 section 解析) ────── 低风险，可先做
    │
步骤 1 (parse_next_turn fallback) ── 核心修复，必须先于路由改动
    │
步骤 2 (移除路由覆盖) ────────── 核心改动，依赖步骤 1
    │
步骤 3 (简化终止逻辑) ────────── 与步骤 2 紧密耦合，一起做
    │
步骤 4 (SharedBoard 全局可见) ── 功能增强，可在步骤 2/3 后做
    │
步骤 7 (清理废弃字段) ────────── 最后清理，依赖前面所有步骤
```

**说明**：步骤 5（初始发言者由 LLM 决定）已移除。初始发言者固定为 Ops，之后的发言完全由 LLM 的 `next_turn` 指令驱动。

### 防止无限辩论的兜底机制

两层安全阀：

**第一层：`max_turns`（默认 12 轮）**

```python
if self._state.current_turn >= self._state.max_turns:
    self._state.terminated = True
    self._state.termination_reason = "max_turns"
    return True
```

**第二层：非 Report 角色 terminate → 强制进入 Report**

```python
elif next_hint == NextTurnHint.TERMINATE:
    if role == DebateRole.REPORT:
        self._state.terminated = True
        self._state.termination_reason = "converged"
        break
    else:
        # 确保最终一定有 Report 发言，方案才能导出
        next_role = DebateRole.REPORT
```

这是唯一保留的编排器覆盖规则——它不否决 LLM 的发言顺序，只确保最后有一位 Report 总结。所有其他硬编码阈值（review_rounds≥5、debate_cycles<2、converge_attempts≥5）都会被移除。

**建议分批**：

| 批次 | 步骤 | 说明 |
|---|---|---|
| 第一批 | 6 → 1 → 2 → 3 | 核心辩论逻辑重构，使辩论真正 LLM 驱动 |
| 第二批 | 4 | Agent 间全局可见性 |
| 第三批 | 7 | 清理废弃字段 |
